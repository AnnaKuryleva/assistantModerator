# Assistant Moderator API

API-сервис для автоматизированной модерации текстового контента с использованием технологий искусственного интеллекта GigaChat.

> [!IMPORTANT]
## Статус проекта: В разработке

Функциональность может расширяться и изменяться.  
Приветствуются отзывы и предложения.

---

## Описание проекта

Проект **Assistant Moderator API** реализуется как backend-сервис (API), который поможет автоматизировать процесс модерации текстового контента на различных платформах (онлайн-чаты, форумы, системы комментариев).

---

##  Используемый стек технологий

На данный момент проект реализован с использованием следующих ключевых технологий и библиотек:

*   **Java 17+:** Основной язык разработки.
*   **Spring Boot 3.2.5:** Фреймворк для создания standalone-приложения.
*   **MapStruct:** Инструмент для генерации типобезопасных классов сопоставления компонентов (bean mapping).
*   **PostgreSQL:** Реляционная база данных для хранения информации о пользователях и истории модерации.
*   **Liquibase:** Инструмент для управления версиями схемы базы данных (миграции).
*   **Jasypt Spring Boot:** Библиотека для шифрования чувствительных данных в конфигурационных файлах (`application.yaml`) с использованием мастер-ключа из переменной окружения.
*   **Springdoc OpenAPI / Swagger UI:** Автоматическая генерация документации API и интерактивный интерфейс для тестирования эндпоинтов.
*   **WebClient (Spring WebFlux):** Для  взаимодействия с внешним API GigaChat.
*   **API:** GigaChat API (OAuth2, REST)
*   **Docker** — контейнеризация приложения
*   **Docker Compose** — оркестрация контейнеров 

#### Ключевые цели и возможности:

*   **Анализ негатива:** Оценка входящих сообщений на предмет наличия грубости, оскорблений, ненормативной лексики.
*   **Многокритериальная оценка:** Предоставление детальной оценки по нескольким критериям (тон, контекст, уровень негатива).
*   **Система отслеживания пользователей:** Отслеживание пользователей, сообщения которых регулярно имеют негативный, грубый, оскорбительный характер ( счётчик негативных сообщений для автоматического предупреждения или блокирования после достижения допустимого лимита).
      * Использование Hibernate `@Formula` для динамического расчета рейтинга токсичности пользователя на стороне БД.
      * Автоматическая блокировка пользователя при накоплении 3-х и более сообщений с критическим уровнем токсичности (score >= 9).
      * Информирование о статусе блокировки непосредственно в ответе API.
    > **Важно:** Подразумевается, что после уведомления о блокировке, пользователь блокируется на стороне клиента без возможности дальнейшей отправки сообщений.
*   **Интеграция с базой данных:** Сохранение истории модерации и статистики пользователей.

### Планы по развитию (Roadmap):

*   **Асинхронная обработка (Kafka):** Переход на асинхронную обработку результатов модерации для повышения производительности и отказоустойчивости.

## Развертывание и запуск

### Предварительные требования

Приложение взаимодействует с внешним сервисом, который использует специфические российские корневые сертификаты (`russian_trusted_root_ca.cer`, `russian_trusted_root_ca_gost_2025.cer`).

Эти сертификаты *не* включены по умолчанию в стандартные дистрибутивы Java. Если вы используете Docker Compose в Docker-образе, то сертификаты будут импортированы автоматически, иначе для успешной работы с API GIGA чат их необходимо добавить в доверенное хранилище Java (Java Keystore / `cacerts`).

## Docker

1. Клонируйте репозиторий

   git clone https://github.com/AnnaKuryleva/assistantModerator

   cd assistant-moderator

2. Подготовьте файл .env

   Cоздайте новый файл .env в корне проекта (рядом с docker-compose.yml).

> **Важно:** Проект поддерживает шифрование чувствительных данных с помощью библиотеки **Jasypt**. Вы можете использовать зашифрованные значения в формате `ENC(...)`, указав мастер-ключ в переменной `JASYPT_ENCRYPTOR_PASSWORD`, либо указывать данные в открытом виде.
> .env файл не должен попадать в открытый репозиторий.

### Данные для подключения

```
DB_URL=jdbc:postgresql://localhost:5433/assistantModerator
DB_USERNAME=admin
DB_PASSWORD=admin
```

#### Ключи GigaChat API  [https://developers.sber.ru/docs/ru/gigachat/api/reference/rest/post-token](https://developers.sber.ru/docs/ru/gigachat/api/reference/rest/post-token)

```
GIGACHAT_UID=ваш_client_id
GIGACHAT_SECRET=ваш_client_secret
GIGACHAT_RQ_UID=6f0b1291-c7f3-43c6-bb2e-9f3efb2dc98e
```

#### Мастер-ключ Jasypt

```
JASYPT_ENCRYPTOR_PASSWORD=ваш_секретный_ключ
```

#### Пароль для инициализации базы в Docker (если планируете запуск через docker-compose)
```
POSTGRES_PASSWORD=admin
```

```

├── docker-compose.yml
├── .env              ← файл
```

## Запуск без Docker

Вы можете скачать сертификаты с:  
🔗 [https://www.sberbank.ru/ru/certificates](https://www.sberbank.ru/ru/certificates)

Вам нужно добавить файлы сертификатов, расположенные в папке `certificates/`, в системное хранилище `cacerts` вашей локальной JDK с помощью утилиты `keytool`.

#### 1. Найдите `keytool`

Утилита `keytool` находится в директории `bin` установленной JDK. Путь обычно выглядит так:

*   **Windows:**  
    `%JAVA_HOME%\bin\keytool.exe`  
    или  
    `C:\Program Files\Java\jdk-21\bin\keytool.exe`

*   **macOS / Linux:**  
    `$JAVA_HOME/bin/keytool`  
    или  
    `/usr/lib/jvm/java-21-openjdk/bin/keytool`

#### 2. Выполните команды импорта

Откройте терминал (командную строку), перейдите в корневую папку проекта и выполните команды.  
**Замените `%JAVA_HOME%` на актуальный путь к вашей JDK**, если переменная окружения не настроена.

> **Стандартный пароль для хранилища `cacerts` по умолчанию — `changeit`**


### Команда для первого сертификата

```bash
"%JAVA_HOME%\bin\keytool.exe" -importcert -cacerts -storepass changeit -file "certificates/russian_trusted_root_ca.cer" -alias RussianTrustedCA1 -noprompt
```
### Команда для второго сертификата

```bash
"%JAVA_HOME%\bin\keytool.exe" -importcert -cacerts -storepass changeit -file "certificates/russian_trusted_root_ca_gost_2025.cer" -alias RussianTrustedCA2025 -noprompt
```

---

### Запуск

1. Убедитесь, что сертификаты добавлены.
2. Настройте ваши учетными данные.
3. Соберите и запустите приложение:

```bash
./mvnw clean package
java -jar target/assistant-moderator.jar
```

## Использование API

Доступен эндпоинт:

### `POST /api/moderation/checkText`

Модерация текста.

**1. Тело запроса:**

```json
{
  "userId": 17,
  "userText": "Мороз и солнце; день чудесный!Еще ты дремлешь, друг прелестный — Пора, красавица, проснись!"
}
```

**Пример ответа:**

```json
{
  "negative": false,
  "score": 0,
  "context": "Поэтичное описание зимней природы и пробуждения",
  "tone": "Положительный, романтический",
  "userBlocked": false
}
```

---
**2. Тело запроса:**
```json
{
  "userId": 1,
  "userText": "здесь_был_оскорбительный_текст"
}
```

**Пример ответа:**

```json
{
  "negative": true,
  "score": 9,
  "context": "грубое выражение неприязни и угрозы [ПОЛЬЗОВАТЕЛЬ ЗАБЛОКИРОВАН]",
  "tone": "агрессивная и угрожающая",
  "userBlocked": true
}
```
---
### `GET /api/users/blocked`

**Пример ответа:**
> **Важно:** Так как в режиме тестирования сервис принимает сообщения и после уведомления о блокировке, то "negativeCount" может принимать значение  > 3

```json
[
  {
    "id": 0,
    "negativeCount": 4,
    "blocked": true
  },
  {
    "id": 1,
    "negativeCount": 7,
    "blocked": true
  }
]
```
