# Assistant Moderator API

API-сервис для автоматизированной модерации текстового контента с использованием технологий искусственного интеллекта GigaChat.

## Описание проекта

Проект **Assistant Moderator API** задумывается как backend-сервис (API), который поможет автоматизировать процесс модерации текстового контента на различных платформах (онлайн-чаты, форумы, системы комментариев).

### Ключевые цели и возможности (Roadmap):

*   **Анализ негатива:** Оценка входящих сообщений на предмет наличия грубости, оскорблений, ненормативной лексики.
*   **Многокритериальная оценка:** Предоставление детальной оценки по нескольким критериям (тон, контекст, уровень негатива).
*   **Система отслеживания пользователей:** Внедрение логики для отслеживания пользователей сообщения которых регулярно имеют негативный, грубый, оскорбительный характер.(например, счетчик негативных сообщений для автоматического предупреждения или блокирования после достижения допустимого лимита).
*   **Интеграция с базой данных:** Сохранение истории модерации и статистики пользователей (планируется использование PostgreSQL).
*   **Асинхронная обработка (Kafka):** Переход на асинхронную обработку результатов модерации для повышения производительности и отказоустойчивости (планируется).

## Инструкции по запуску 



### Предварительные требования

Приложение взаимодействует с внешним сервисом, который использует специфические российские корневые сертификаты (`russian_trusted_root_ca.cer`, `russian_trusted_root_ca_gost_2025.cer`).

Эти сертификаты *не* включены по умолчанию в стандартные дистрибутивы Java. Для успешной работы с API GIGA чат их необходимо добавить в доверенное хранилище Java (Java Keystore/`cacerts`).

Вы можете скачать сертификаты с https://www.sberbank.ru/ru/certificates

Вам нужно добавить файлы сертификатов, расположенные в папке `certificates/`, в системное хранилище `cacerts` вашей локальной JDK с помощью утилиты `keytool`.

1.  **Найдите `keytool`:**
    `keytool` находится в директории `bin` вашей установленной JDK. Путь к ней обычно выглядит так:
    *   **Windows:** `%JAVA_HOME%\bin\keytool.exe` или `C:\Program Files\Java\jdk-21\bin\keytool.exe`
    *   **macOS/Linux:** `$JAVA_HOME/bin/keytool` или `/usr/lib/jvm/java-21-openjdk/bin/keytool`

2.  **Выполните команды импорта:**
    Откройте терминал (командную строку), перейдите в корневую папку проекта и выполните следующие команды. **Замените `%JAVA_HOME%` на ваш актуальный путь** к папке JDK, если переменная окружения не установлена.

    *(Стандартный пароль для хранилища `cacerts` по умолчанию в большинстве JDK — `changeit`)*

    ```bash
    # Команда для первого сертификата
    "%JAVA_HOME%\bin\keytool.exe" -importcert -cacerts -storepass changeit -file "certificates/russian_trusted_root_ca.cer" -alias RussianTrustedCA1 -noprompt

    # Команда для второго сертификата
    "%JAVA_HOME%\bin\keytool.exe" -importcert -cacerts -storepass changeit -file "certificates/russian_trusted_root_ca_gost_2025.cer" -alias RussianTrustedCA2025 -noprompt
    ```
